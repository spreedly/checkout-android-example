# Migration Guide: 0.11.x to 0.12.0

Version 0.12.0 has one breaking change: the `sdkPlatform` parameter on `SpreedlySDKInitOptions` changed from a `String` to a type-safe `SdkPlatform` enum. This guide covers what to do. See the [CHANGELOG](../../CHANGELOG.md) for the full release notes.

## 1. `sdkPlatform` Is Now a `SdkPlatform` Enum

`SpreedlySDKInitOptions.sdkPlatform` was a `String` (default `"android"`). It is now a `SdkPlatform` enum with two entries:

| Enum value | `value` property | Use case |
|---|---|---|
| `SdkPlatform.ANDROID` | `"checkout-android"` | Native Android apps (default) |
| `SdkPlatform.REACT_NATIVE` | `"checkout-react-native"` | React Native bridge integrations |

The enum's `value` is sent as both the Datadog `sdk_platform` global attribute and the `source` field on payment method creation requests to Core.

### Native Android callers

If you rely on the default, no change is needed -- the default shifted from `"android"` to `SdkPlatform.ANDROID` automatically.

If you were passing the string explicitly:

```diff
 SpreedlySDKInitOptions(
     nonce = nonce,
     signature = signature,
     certificateToken = certToken,
     timestamp = timestamp,
     environmentKey = envKey,
-    sdkPlatform = "android",
+    // default is SdkPlatform.ANDROID -- just drop the parameter
 )
```

### React Native bridge callers

```diff
 SpreedlySDKInitOptions(
     nonce = nonce,
     signature = signature,
     certificateToken = certToken,
     timestamp = timestamp,
     environmentKey = envKey,
-    sdkPlatform = "react_native",
+    sdkPlatform = SdkPlatform.REACT_NATIVE,
 )
```

Import: `import com.spreedly.sdk.SdkPlatform`

## Non-Breaking Changes

- **`source` field on payment method creation**: All payment method creation requests (credit card and offsite/APM) now automatically include a `source` field identifying the SDK and platform. No merchant action required.

## Verification Checklist

- [ ] Build succeeds: `./gradlew assembleRelease`
- [ ] No raw string `sdkPlatform = "react_native"` or `sdkPlatform = "android"` in app code -- zero results
- [ ] End-to-end payment flow works
- [ ] Datadog logs show correct `sdk_platform` and `sdk_version` attributes (if Datadog is enabled)
