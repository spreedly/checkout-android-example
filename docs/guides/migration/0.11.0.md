# Migration Guide: 0.10.4 to 0.11.0

Version 0.11.0 has four breaking changes: 3DS extracted into a standalone module, 3DS packages renamed, offsite deep link scheme moved into the SDK, and `SavedCardInfo.cardBrand` renamed. This guide lists what to do for each. See the [CHANGELOG](../../CHANGELOG.md) for the full release notes.

## 1. 3DS Dependency Moved

3DS support moved from `checkout-payments-core` to a new `checkout-threeds` module. The Forter SDK is now a transitive dependency of `checkout-threeds`.

**Action (if using 3DS):**

- Add `implementation("com.spreedly:checkout-threeds:0.11.0")` to your Gradle dependencies
- Remove any explicit Forter SDK dependency
- The module auto-registers via `ThreeDSInitProvider` -- no manual setup needed

See the [3DS Global Guide](../3ds-global.md) for full dependency setup and Forter Maven repository configuration.

**If you do not use 3DS**, no action is needed.

## 2. 3DS Package Rename

All 3DS implementation classes moved from `com.spreedly.sdk.threeds` to `com.spreedly.threeds`. Update imports in your code.

| 0.10.4 | 0.11.0 |
|--------|--------|
| `com.spreedly.sdk.threeds.gatewayspecific.GatewaySpecific3DSIntegration` | `com.spreedly.threeds.gatewayspecific.GatewaySpecific3DSIntegration` |
| `com.spreedly.sdk.ui.threeds.ThreeDSChallengeSheet` | `com.spreedly.threeds.ui.ThreeDSChallengeSheet` |
| `com.spreedly.sdk.ui.threeds.ThreeDSChallengeContent` | `com.spreedly.threeds.ui.ThreeDSChallengeContent` |
| `com.spreedly.sdk.ui.threeds.ThreeDSChallengeBottomSheet` | `com.spreedly.threeds.ui.ThreeDSChallengeBottomSheet` |

The `ThreeDSProvider` interface remains at `com.spreedly.sdk.threeds.ThreeDSProvider` in `payments-core` (unchanged). The public API on `Spreedly` (`showThreeDSChallenge()`, `hideThreeDSChallenge()`, flows, state) is also unchanged.

## 3. Offsite Deep Link Scheme

The SDK now owns `OffsiteReturnActivity` with scheme `${applicationId}.spreedlyoffsite`. Merchants no longer define their own return activity.

**Action:**

1. Remove your custom `OffsiteReturnActivity` class and its manifest entry:

```diff
 <application>
-    <activity
-        android:name=".OffsiteReturnActivity"
-        android:exported="true"
-        android:launchMode="singleTask">
-        <intent-filter>
-            <action android:name="android.intent.action.VIEW" />
-            <category android:name="android.intent.category.DEFAULT" />
-            <category android:name="android.intent.category.BROWSABLE" />
-            <data android:scheme="spdlapp" />
-        </intent-filter>
-    </activity>
 </application>
```

2. Update your backend `redirect_url` to use the new scheme. The SDK provides helpers to get the correct value at runtime -- see the [Offsite Payments Guide](../offsite-payments.md) for `SpreedlyOffsiteCheckout.redirectUrl(context)` and `deepLinkScheme(context)` usage.

| | Scheme | Example URL |
|-|--------|-------------|
| **0.10.4** | `spdlapp` | `spdlapp://checkout` |
| **0.11.0** | `${applicationId}.spreedlyoffsite` | `com.myapp.spreedlyoffsite://checkout` |

## 4. `SavedCardInfo.cardBrand` Renamed to `cardholderName`

The field was only ever used for cardholder names; `cardType` already carries the brand.

**Kotlin:**

```diff
 SavedCardInfo(
     lastFourDigits = "4242",
     cardType = "visa",
-    cardBrand = "Jane Doe",
+    cardholderName = "Jane Doe",
 )
```

**Java (`RecacheJavaHelper`):** The 3rd named parameter on `createRecacheConfig` and `createRecacheConfigFull` is renamed from `cardBrand` to `cardholderName`. Positional Java callers are unaffected.

See the [Recaching Guide](../recaching.md) for full `SavedCardInfo` and `RecacheJavaHelper` usage.

## Non-Breaking Changes

- Datadog telemetry added (zero merchant configuration). See [Datadog Integration](../../development/DATADOG_INTEGRATION.md).
- `HostedFieldInteraction` log level raised from `DEBUG` to `INFO`.
- Stripe APM `ApmCheckoutCompleted` emission moved to after `pollForFinalStatus`.
- APM timing race condition fixed with `FlowDurationTracker`.
- Raw payment tokens sanitized from logs via `LogSanitizer.sanitizeUrl()`.
- Fat AAR build task removed (deprecated since 0.7.1).
- OWASP Dependency Check scan removed (Dependabot provides coverage).
